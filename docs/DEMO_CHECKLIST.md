# TechNote 演示验收清单

本文档用于项目展示、交接和手动验收。它只描述需要手动执行和观察的步骤，不代表已经在当前环境运行过。

## 1. 环境确认

当前项目按以下本机版本设计：

- Java 21.0.9 LTS
- Spring Boot 3.3.7
- MySQL 8.0.44
- Node 24.11.0
- npm 11.6.1

验收前先确认：

- 后端配置中的 MySQL 地址、账号、密码和数据库名可用。
- 文件上传目录有写入权限。
- 前端 API 基础地址指向当前后端服务。
- `technote.public-site-url` 指向当前前端站点地址，用于 RSS 和 sitemap 生成公开链接。
- 当前数据库是演示库或可安全导入演示数据的本地库。

## 2. 数据库准备

全新数据库建议顺序：

```text
database/technote_schema.sql
database/seed_demo_data.sql
```

已有数据库按需补升级脚本：

```text
database/upgrade_article_interaction.sql
database/upgrade_dashboard_statistics.sql
database/upgrade_link_apply.sql
database/upgrade_public_submit_guard.sql
database/seed_demo_data.sql
```

导入后检查：

- `sys_user` 存在 `admin` 账号。
- `blog_article` 存在 4 篇公开文章和 1 篇草稿演示文章。
- `blog_article_view_daily` 存在阅读趋势数据。
- `blog_comment`、`blog_guestbook`、`blog_link_apply` 存在待审核演示数据。
- `blog_guard_setting` 存在公开提交风控配置。

## 3. 构建与启动

这些命令需要在获得允许后手动运行：

```text
cd backend
mvn spring-boot:run
```

```text
cd frontend
npm install
npm run dev
```

构建验收建议：

- 后端启动无 Mapper、配置绑定、表字段映射错误。
- 前端依赖安装无版本冲突。
- 前端页面打开后控制台无首屏阻塞性报错。
- 如果执行 `npm run build`，应确认 TypeScript 和 Vite 构建通过。

## 4. 公开站点验收

首页：

- 能看到演示文章列表。
- 分类、专栏、标签筛选可用。
- 热门文章侧栏有数据。
- 列表为空或请求失败时显示统一空状态或错误提示。
- 浏览器标题、description、canonical 和 RSS alternate 指向当前站点。
- 页面 `<head>` 中存在 `script#technote-json-ld[type="application/ld+json"]`，内容为 WebSite，并包含 SearchAction 搜索入口。

文章详情：

- Markdown 标题、代码块、行号和复制按钮显示正常。
- 目录锚点可跳转。
- 相关文章展示正常。
- 点赞和取消点赞不会重复计数异常。
- 复制链接按钮能复制当前文章公开地址；支持 Web Share API 的浏览器可打开系统分享面板，不支持时退回复制链接。
- 评论区支持两层评论展示。
- 评论提交需要验证码；验证码错误、敏感词和频率限制应有明确提示。
- 文章详情页的浏览器标题、description、Open Graph 和 canonical 使用当前文章标题、摘要和 slug。
- 文章详情页 `<head>` 中的 JSON-LD 为 BlogPosting，包含 headline、url、datePublished、dateModified、author 和 keywords。

归档：

- 月份归档有文章。
- 年度写作热力图在 2026 年有演示数据。
- 热力图只统计公开已发布文章，不展示草稿或私密内容。

搜索：

- 搜索 `Spring`、`Vue`、`Markdown` 能返回对应文章。
- 搜索结果不返回完整正文。
- 关键词高亮不依赖不安全 HTML 注入。

专栏：

- 专栏列表能展示 `全栈博客从 0 到 1` 和 `持续构建日志`。
- 专栏详情能按专栏内排序展示文章。

友链和留言：

- 友链列表展示 Spring、Vue、shadcn-vue。
- 友链申请表单带验证码。
- 留言板展示已审核留言，待审核留言不在公开侧展示。

关于与 404：

- 关于页能读取站点设置内容。
- 未知地址进入 404 页面，并能返回首页或搜索页。

RSS、sitemap 与 robots：

- `/api/rss.xml` 返回 RSS 2.0 XML，频道标题、简介和文章链接正确。
- `/api/sitemap.xml` 返回 sitemap XML，包含首页、归档、专栏、友链、留言、关于页和公开文章。
- `/api/robots.txt` 返回纯文本，允许公开站点抓取、禁止后台路径，并声明 sitemap 地址。
- XML 中的文章链接使用前端路径 `/articles/{slug}`，域名来自 `technote.public-site-url`。
- 公开侧栏展示 RSS、sitemap 和 robots 入口，点击后能打开对应资源。
- 页面 `<head>` 中存在 `link rel="alternate"`，类型为 `application/rss+xml`。
- 在公开普通页面和文章详情页之间切换时，JSON-LD 内容应随路由更新，不能残留上一篇文章的 BlogPosting。

## 5. 后台验收

登录：

- 使用默认管理员账号登录。
- 登录后刷新页面仍能保持会话。
- 退出后访问后台路由会回到登录页。

数据概览：

- 统计卡片显示文章、草稿、阅读、待审互动等数据。
- 近 7/30 天趋势图有演示阅读数据。
- 分类占比、热门排行和最近操作有内容。

文章管理：

- 列表显示公开文章和草稿，状态、分类、专栏、标签可读。
- 新建、编辑、保存、预览 Markdown 流程可用。
- 自动草稿能保存、恢复和清理。
- 版本列表、版本详情和回滚可用。
- 批量设公开/私密、批量改分类、批量移入回收站可用。
- 单篇和批量 Markdown 导出可用。

内容组织：

- 分类、标签、专栏 CRUD 可用。
- 专栏文章关联、排序和移除可用。
- 删除被文章使用的专栏时应有明确保护或提示。

互动审核：

- 评论审核列表能看到待审核评论。
- 留言审核列表能看到待审核留言。
- 友链申请列表能看到待审核申请。
- 审核通过后公开侧能展示对应内容，驳回后公开侧不展示。

站点与风控：

- 站点设置保存后公开页能读取更新。
- 公开风控配置页能调整验证码、TTL、敏感词和提示语。
- 关闭验证码后，公开评论/留言/友链申请不再强制输入验证码。

媒体、回收站和日志：

- 图片上传后能在媒体库看到记录。
- 删除文章进入回收站，恢复后文章可再次使用。
- 永久删除会清理文章相关关系和统计数据。
- 后台操作日志能记录关键操作，并脱敏密码字段。

个人中心：

- 能更新昵称、头像、邮箱和简介。
- 修改密码需要旧密码和确认密码。
- 修改密码成功后当前会话退出，需要重新登录。

## 6. 响应式与可访问性

桌面端：

- 后台侧栏滚动不影响主内容。
- 表格横向内容在表格内部滚动，不撑破整页。
- 弹窗表单、编辑器、图表和长文本不互相遮挡。

移动端：

- 公开站点顶部导航和筛选入口可横向滚动。
- 文章目录在正文前展示，不遮挡标题。
- 后台使用抽屉导航，切换路由后抽屉自动关闭。
- 表格、筛选区和操作按钮在窄屏下不重叠、不丢失。

状态反馈：

- 加载态、错误态、空状态互斥显示。
- 提交按钮在请求中禁用，避免重复提交。
- 表单错误信息靠近对应输入区域。

## 7. 回归风险重点

- 公开查询必须过滤 `deleted = 0`、`status = 1`、`visibility = 1`。
- 后台写操作应保持登录校验和操作日志。
- 评论、留言、友链申请应保持限流、验证码和敏感词顺序。
- 点赞、阅读趋势、审核通过这类计数和状态变更要关注事务与幂等。
- RSS、sitemap 和 robots 只能指向公开站点资源，不能暴露后台管理接口或草稿、私密、回收站内容。
- 演示数据脚本可重复执行，但不要在生产库导入。
